## 深拷贝和浅拷贝

### 1. 浅拷贝

浅拷贝只复制对象的第一层属性。如果属性是基本类型，复制的就是**值**；如果属性是引用类型（对象或数组），复制的就是**内存地址**。

```js
// 基本类型 - 直接赋值就是拷贝值
let a = 10
let b = a
b = 20
console.log(a) // 10 ✅ 互不影响

// 引用类型 - 直接赋值只是拷贝引用
let obj1 = { name: '张三' }
let obj2 = obj1
obj2.name = '李四'
console.log(obj1.name) // '李四' ❌ 相互影响
```

- **后果：** 对于引用类型，拷贝后的新对象和原对象**共享**内部的子对象。修改其中一个，另一个也会跟着变。

- **常用方法：**

  - `Object.assign()`
  - 展开运算符 `{...obj}`
  - `Array.prototype.slice()`

  ```js
  const original = {
    name: '张三',
    age: 18,
    address: {
      city: '北京',
      district: '朝阳区'
    },
    hobbies: ['读书', '跑步']
  }
  
  const shallowCopy = { ...original }
  
  // 修改第一层属性 - 不影响原对象
  shallowCopy.name = '李四'
  shallowCopy.age = 20
  console.log(original.name)  // '张三' ✅
  console.log(original.age)   // 18 ✅
  
  // 修改第二层引用类型 - 影响原对象
  shallowCopy.address.city = '上海'
  shallowCopy.hobbies.push('游泳')
  console.log(original.address.city)  // '上海' ❌
  console.log(original.hobbies)       // ['读书', '跑步', '游泳'] ❌
  ```

- **手动实现浅拷贝**

  ```js
  function shallowClone(obj) {
    if (typeof obj !== 'object' || obj === null) return obj
    
    const newObj = Array.isArray(obj) ? [] : {}
    
    for (let key in obj) {
      // 检查一个属性是否属于对象“自己”，而不是从“原型链”上继承来的
      if (obj.hasOwnProperty(key)) {
        newObj[key] = obj[key]  // 直接赋值
      }
    }
    
    return newObj
  }
  
  // 测试
  const original = {
    a: 1,
    b: { c: 2 }
  }
  const copy = shallowClone(original)
  copy.b.c = 3
  console.log(original.b.c)  // 3 ❌
  ```

### 2. 深拷贝 

深拷贝会递归地复制对象的所有层级。它会在堆内存中开辟一块全新的空间，把原对象的所有内容（包括嵌套的子对象）完整地克隆一份。

- **后果：** 两个对象完全**独立**，互不影响。修改新对象不会对原对象产生任何副作用。

- **常用方法：**

  - **简单粗暴法：** `JSON.parse(JSON.stringify(obj))`
    - *缺点：* 
      - 无法复制函数、undefined、Symbol
      - 无法复制 Date、RegExp、Map、Set 等特殊对象
      - 无法处理循环引用
      - 会忽略原型链上的属性


  - **现代标准法：** `structuredClone(obj)` (原生支持，推荐)
  - **库函数：** Lodash 的 `_.cloneDeep()`

  ```js
  const original = {
    name: '张三',
    age: 18,
    address: {
      city: '北京',
      district: '朝阳区'
    },
    hobbies: ['读书', '跑步'],
    sayHi: function() {
      console.log('你好')
    },
    date: new Date(),
    reg: /test/g,
    undefined: undefined,
    null: null,
    symbol: Symbol('test')
  }

  const deepCopy = JSON.parse(JSON.stringify(original))

  // 验证完全独立
  deepCopy.address.city = '上海'
  deepCopy.hobbies.push('游泳')

  console.log(original.address.city)  // '北京' ✅
  console.log(original.hobbies)       // ['读书', '跑步'] ✅

  // ❌ JSON 方法的局限性：
  console.log(deepCopy.sayHi)         // undefined - 函数丢失
  console.log(deepCopy.date)          // 字符串，不是 Date 对象
  console.log(deepCopy.reg)           // {} - 正则表达式丢失
  console.log(deepCopy.undefined)     // 丢失
  console.log(deepCopy.symbol)        // 丢失
  console.log(deepCopy)               // 不包含函数、undefined、symbol
  ```

#### 正则表达式的拷贝

正则表达式包含**模式**（pattern）和**标志**（flags），需要完整复制这两部分。

```js
// 正则的组成部分
const regex = /abc/gi
console.log(regex.source)      // 'abc'（source：正则表达式的模式文本）
console.log(regex.flags)       // 'gi'（flags：标志字符串）
console.log(regex.global)      // true
console.log(regex.ignoreCase)  // true
console.log(regex.multiline)   // false

// 正则的构造函数形式
const regex2 = new RegExp('abc', 'gi')
console.log(regex2.source)  // 'abc'
console.log(regex2.flags)   // 'gi'
```



```js
// 方法1：使用 source 和 flags
function copyRegex1(regex) {
  return new RegExp(regex.source, regex.flags)
}

// 方法2：使用构造函数（直接传递正则对象）
function copyRegex2(regex) {
  return new RegExp(regex)  // ES6+ 支持
}

// 方法3：手动提取标志
function copyRegex3(regex) {
  const flags = []
  if (regex.global) flags.push('g')
  if (regex.ignoreCase) flags.push('i')
  if (regex.multiline) flags.push('m')
  if (regex.dotAll) flags.push('s')
  if (regex.unicode) flags.push('u')
  if (regex.sticky) flags.push('y')
  
  return new RegExp(regex.source, flags.join(''))
}

// 方法4：使用 toString() 解析（不推荐，但了解原理）
function copyRegex4(regex) {
  const str = regex.toString()  // '/abc/gi'
  const lastSlash = str.lastIndexOf('/')
  const pattern = str.slice(1, lastSlash)
  const flags = str.slice(lastSlash + 1)
  return new RegExp(pattern, flags)
}

// 测试
const original = /abc/gi
const copy1 = copyRegex1(original)
const copy2 = copyRegex2(original)

console.log(copy1.source === original.source)  // true
console.log(copy1.flags === original.flags)    // true
console.log(copy1 === original)                // false（不同对象）
```

// 递归
function deepClone3(obj, map = new WeakMap()) {
   if (obj === null || typeof obj !== 'object') return obj;
   if (map.has(obj)) return map.get(obj); // 避免无限循环， 如果这个对象已经拷贝过，直接返回已经拷贝过的对象

   const clone = Array.isArray(obj) ? [] : {};

       clone[key] = deepClone3(obj[key], map);
   });
   return clone;
}

### 3. 形象的比喻

想象你有一张**藏宝图**（对象），地图上标记了一个**宝箱的位置**（子对象）。

- **浅拷贝：** 你复印了一张地图。现在有两张地图，但它们指向的是**同一个宝箱**。如果有人去把宝箱里的金子换成了石头，两张地图的使用者都会发现宝箱变了。
- **深拷贝：** 你不仅复印了地图，还**克隆了一个一模一样的宝箱**放在另一个地方。现在你有两张地图和两个独立的宝箱，你在其中一个箱子里放火，另一个箱子安然无恙。

### 4.场景对比表

| **特性**     | **浅拷贝**             | **深拷贝**           |
| ------------ | ---------------------- | -------------------- |
| **复制深度** | 仅限第一层             | 无限层级（递归）     |
| **内存空间** | 与原对象共享子对象地址 | 开辟完全独立的新空间 |
| **性能**     | 快                     | 慢（尤其是大对象）   |
| **修改影响** | 修改嵌套对象会相互影响 | 完全互不影响         |

## 防抖

### 1.什么是防抖

防抖是一种**限制函数执行频率**的技术，确保在**连续触发**的事件中，函数只在最后一次触发后的指定时间间隔后执行一次。

- 频繁触发的操作，只关心最后一次结果
- 比如：输入框搜索、窗口resize、按钮提交

### 2.代码实现

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>
    <button>按钮点击(普通)</button>
    <input id="search" type="text" placeholder="请输入搜索内容">
    <br>
    
    <button id="btn-immediate">按钮点击 (立即执行版)</button>
    <button id="btn-cancel">取消上述按钮的等待执行</button>
    <br>
    <input id="search2" type="text" placeholder="请输入搜索内容 (非立即执行)">

    <script>
        // 防抖：是一种限制函数执行频率的技术，确保在连续触发的事件中，函数只在最后一次触发后的指定时间间隔后执行一次。
        
        // 1 基础版防抖函数
        function debounce1(fn, delay) {
            let timer = null; // 闭包变量
            // debounce(fn,300)时，fn 并没有执行
            // 这里返回的函数 会执行console回调
            return function(...args) {
                clearTimeout(timer);
                timer = setTimeout(() => fn.apply(this, args), delay);
                // apply方法允许手动指定函数内部的this是谁，同时以数组的形式传入参数
                // 用 button 作为 this，把事件对象传给原函数
            };
        }

        // 按钮防抖
        const button = document.querySelector('button')
        button.addEventListener('click', debounce1(() => console.log('点击了按钮'), 300));
        
        // 输入框实时搜索
        const search = document.querySelector('#search')
        const debounceSearch = debounce1(()=>{
            console.log('输入框内容变化',search.value);
        },300)
        search.addEventListener('input',debounceSearch)

        // 窗口resize
        const debounceResize = debounce1(() => {
            console.log('窗口大小变化了');
        },300)
        window.addEventListener('resize',debounceResize)

// -------------------------------------------------------------------------------------------------        

        // 2  带有 immediate 参数的防抖函数（完整版）
        // 开启立即执行：点击立刻生效，连续狂点无效，停手 delay 秒后才能再次点击）
        // 非立即执行：打字停顿 delay秒 后再请求
        function debounce2(fn, delay = 300, immediate = false) {
            let timer = null;
            // 这里是真正被绑定的事件回调函数
            const _debounce = function(...args) {
                // 每次触发都先清除之前的定时器，重新计时
                if (timer) clearTimeout(timer);
                if (immediate) {
                    // 【立即执行逻辑】
                    // 如果 timer 为空，说明是“第一次”触发，或者上一次触发的锁定时间已经过了
                    const callNow = !timer;
                    // 开启一个定时器，等待 delay 时间后，将 timer 清空，解除锁定
                    timer = setTimeout(() => {
                        timer = null;
                    }, delay);

                    // 如果判断应该立即执行，就立刻调用原函数
                    if (callNow) {
                        fn.apply(this, args);
                    }
                } else {
                    // 【非立即执行逻辑】（也就是你原本的基础版）
                    timer = setTimeout(() => {
                        fn.apply(this, args);
                    }, delay);
                }
            };

            // 【取消功能】给返回的函数挂载一个 cancel 方法：作用仅仅是“中断当前这一次的定时器”。
            // 对于 immediate = true 的立即执行版防抖来说，调用 cancel() 会把 timer 清空，
            // 这意味着你的下一次点击会立刻执行，但点击完之后，它又会重新开启新的定时器，继续防抖。
            _debounce.cancel = function() {
                if (timer) clearTimeout(timer);
                timer = null;
                console.log('防抖已被手动取消');
            };

            return _debounce;
        }

        // 1. 按钮防抖（开启立即执行：点击立刻生效，连续狂点无效，停手 1 秒后才能再次点击）
        const btnImmediate = document.querySelector('#btn-immediate');
        const handleBtnClick = debounce2(() => {
            console.log('🚀 按钮触发了操作！', new Date().toLocaleTimeString());
        }, 5000, true); 
        btnImmediate.addEventListener('click', handleBtnClick);

        // 3. 输入框实时搜索（非立即执行：打字停顿 500ms 后再请求）
        const search2 = document.querySelector('#search2');
        const handleSearch = debounce2(function() {
            console.log('🔍 搜索请求发送:', this.value);
        }, 500, false);
        search2.addEventListener('input', handleSearch);        

        // 3. 取消防抖（只对“非立即执行”有用，或者在立即执行的锁定期间内解除锁定）
        const btnCancel = document.querySelector('#btn-cancel');
        btnCancel.addEventListener('click', () => {
            handleBtnClick.cancel();
        });
    </script>
</body>
</html>
```

####  `return function(...args)`：返回包装函数并收集参数

- **为什么要 `return function`？** `debounce` 是一个高阶函数（接受函数作为参数，并返回一个新函数的函数）。当你执行 `debounce(fn, 300)` 时，`fn` 并没有执行。`debounce` 的作用是制造并**返回一个新的函数**。这个新返回的函数，才是真正交给 `button.addEventListener` 去执行的回调函数。
- **`...args` 是什么意思？** 这是 ES6 的剩余参数（Rest parameters）语法。在事件监听器中，当事件触发时，浏览器会自动传入事件对象（比如鼠标点击产生的 `MouseEvent` 对象）。`...args` 的作用是把所有接收到的参数“打包”成一个名为 `args` 的数组。
  - *举个例子：* 如果你不写 `...args`，原函数就拿不到那个包含了鼠标坐标 `clientX / clientY` 的事件对象了。

#### `fn.apply(this, args)`：恢复上下文并执行原函数

当定时器时间到了，终于要执行你最初传入的那个函数 `fn` 时，为了不改变它原本的运行状态，代码使用了 `apply` 方法。

- **为什么要用 `apply`？** `apply` 是 JavaScript 函数自带的一个方法，它的作用是：调用函数，并且允许你手动指定函数内部的 `this` 是谁，同时以数组的形式传入参数。
- **为什么要绑定 `this`？** 在普通的事件绑定中，回调函数内部的 `this` 默认指向触发事件的那个 DOM 元素（在这里就是 `<button>`）。 由于我们是在 `setTimeout` 的回调里执行 `fn` 的，如果直接写 `fn()`，在非严格模式下，`this` 就会丢失，变成指向全局对象 `window`。 *(注：这里 `setTimeout` 里面用的是箭头函数 `() =>`，箭头函数没有自己的 `this`，所以它会向外层找，正好找到了 `return function(...args)` 这里的 `this`，也就是那个 `<button>` 元素。然后我们用 `apply` 把这个正确的 `<button>` 元素作为 `this` 塞进 `fn` 里。)*
- **为什么要传入 `args`？** 把上面 `...args` 收集到的事件对象数组，拆解开原封不动地喂给 `fn`。

## 节流

节流的核心思想是：**规定在一个单位时间内，只能触发一次函数。如果这个单位时间内触发多次函数，只有一次生效。**

### 1.代码实现

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>
    <script>
        // 如果你持续触发事件，每隔一段时间，只执行一次事件。

        // 1.时间戳版
        function throttleByTimestamp(fn, delay) {
            // 记录上一次执行的时间戳
            let lastTime = 0; 
            return function(...args) {
                // 获取当前时间戳
                const nowTime = Date.now(); 
                // 如果当前时间减去上一次执行时间，大于设定的冷却时间 delay
                if (nowTime - lastTime >= delay) {
                    // 执行函数，绑定 this 和 参数
                    fn.apply(this, args);
                    // 更新“上一次执行时间”为当前时间
                    lastTime = nowTime; 
                }
            };
        }
        // 使用：
        window.addEventListener('resize', throttleByTimestamp(() => console.log('页面滚动了'), 1000));

        // 2.定时器版
        function throttleByTimer(fn, delay) {
            let timer = null; // 相当于冷冻期的锁
            return function(...args) {
                // 如果 timer 有值，说明锁没解开，直接 return 忽略本次操作
                if (timer) return;
                // 上锁并开启倒计时
                timer = setTimeout(() => {
                    fn.apply(this, args); // 倒计时结束，执行核心逻辑
                    timer = null;         // 核心逻辑执行完，把锁解开
                }, delay);
            };
        }
        
        // 场景：无限滚动加载监听
        window.addEventListener('resize', throttleByTimer(checkPosition, 200));

    </script>
</body>
</html>
```

### 3. 节流 (Throttle) vs 防抖 (Debounce) 的对比

| **特性**     | **防抖 (Debounce)**                                          | **节流 (Throttle)**                                          |
| ------------ | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **核心逻辑** | **“打断重置”**。只要你一直触发，我就一直重新计时，直到你停手。 | **“按部就班”**。无论你怎么触发，我都按照固定的节奏执行。     |
| **执行次数** | 连续的疯狂操作最终大概率**只执行 1 次**。                    | 连续的疯狂操作会**执行 N 次**（取决于你持续多久和设定的频率）。 |
| **典型场景** | 1. 搜索框实时联想（打字停顿后发请求） 2. 窗口调整 `resize`（拉伸结束后重新渲染视图） 3. 按钮防连击（避免手抖提交两次表单） | 1. 页面滚动 `scroll`（如无限滚动加载图片） 2. 鼠标移动 `mousemove`（如拖拽元素的实时计算） 3. 射击游戏中的开火键（限制射速） |
